# 📋 기획 문서

## 프로젝트 개요

| 항목 | 내용 |
|------|------|
| 프로젝트명 | TeamMaker |
| 한줄 소개 | 개발자와 비개발자의 역할을 자동으로 배분해 균형 잡힌 팀을 구성해주는 스마트 팀
  빌딩 도구 |
| Pain Point | 수동 배치의 비효율, 각 사람의 역량이 맞게 배정되지 않는 문제 |

## 타겟 사용자

- **주요 타겟**: 제로톤 활동
- **사용자 특성**: 개발자와 비개발자가 섞여있다.
- **시장 규모**: 친바로 확장 예정

## MVP 범위

> 2시간 내에 구현 가능한 핵심 기능만 선정하세요.

- [ ] 개발자와 비개발자를 구별하여 역할을 부여하는 기능: 팀을 구성하는 과정에서 비개발자가 개발 역할에 배정되지 않도록 하는 기능

- [ ] 미배정 인원 처리: 자동으로 3명씩 배분, 나머지 인원은 버리지 않고 일부 팀에 1명씩 추가 배정 (예: 10명 3팀 → 팀1: 4명, 팀2: 4명, 팀3: 3명)

- [ ] 팀원 삭제 기능: 추가된 팀원 이름 옆에 X 버튼을 두어 개별 삭제 가능

- [ ] 팀 구성 다시하기: 결과 화면에서 "다시 섞기" 버튼을 누르면 같은 인원으로 팀을 재배정

- [ ] 팀 구성 결과 카드 화면: 팀 구성 완료 후 팀별로 카드 형태로 결과를 표시, 각 카드에 팀원 이름과 역할(개발/기획/마케팅) 표기

-----------------------------   추가 기능    --------------------------------

- [ ] 중복 이름 입력 방지 및 시각적 피드백: 이미 추가된 이름을 다시 입력하면 입력창이 빨간색으로 표시되고 "이미 등록된 이름입니다" 안내 문구를 노출

- [ ] 비개발자 역할 안내: 비개발자 영역에서 이름을 추가할 때 역할을 기획/마케팅 중에서만 선택할 수 있도록 드롭다운 제공, "비개발자는 기획 또는 마케팅 역할만 선택할 수 있습니다" 안내 문구 노출

- [ ] 인원 현황 실시간 표시: 개발자 N명 / 비개발자 N명을 각 영역 하단에 실시간으로 표시, 전체 인원 합계도 함께 노출




## 타임라인

| 시간 | 활동 | 담당 |
|------|------|------|
| 0:00 - 0:15 | 아이디어 확정 & 역할 분담 | 전체 |
| 0:15 - 0:30 | 기획 문서 작성 & 화면 설계 | 기획 |
| 0:30 - 1:30 | 핵심 기능 개발 | 개발 |
| 1:30 - 1:45 | 통합 테스트 & 버그 수정 | 전체 |
| 1:45 - 2:00 | 발표 준비 | 전체 |

## 비즈니스 모델

- **수익 모델**: 초기 무료 제공 → 팀 구성 횟수/인원 제한 기반 프리미엄 플랜 (해커톤 주최 단체 대상 B2B SaaS)
- **차별점**: 단순 랜덤 배치가 아닌 개발자/비개발자 역할을 고려한 균형 잡힌 팀 자동 구성
- **확장 가능성**: 제로톤 → 친바 → 기업 내부 프로젝트 팀 빌딩으로 확장, 역량 태그 기반 매칭 고도화
